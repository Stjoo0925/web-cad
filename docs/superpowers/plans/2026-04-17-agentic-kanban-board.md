# 온프렘 CAD + 포인트클라우드 SDK 에이전틱 칸반 보드

> 목적: 기존 티켓을 기능 나열형이 아니라 `실행 우선순위 + 의존성 + 리스크` 기준으로 재정렬하고, 에이전틱 코딩이 바로 가능한 보드로 운영한다.

## 운영 원칙

- 기준 보드는 이 문서와 GitHub 트래킹 이슈 `#29`를 함께 사용한다.
- 우선순위는 `사용자 가시성`보다 `아키텍처 임계 경로`와 `통합 리스크`를 먼저 본다.
- WIP 제한은 `동시 구현 1개 + 병렬 탐색/리뷰 1개`로 둔다.
- 새 구현 티켓은 반드시 TDD로 시작한다.
- 한 티켓이 끝나면 다음 티켓으로 넘어가기 전에 `테스트 통과`, `문서/보드 갱신`, `다음 티켓 unblock` 여부를 확인한다.

## 우선순위 정렬 기준

1. 실제 화면이 뜨는가
2. 편집기 상태와 뷰포트 구조가 고정됐는가
3. 2D CAD 기본 렌더링이 가능한가
4. DXF와 포인트클라우드가 공통 문서 모델에 들어오는가
5. 편집/협업/레이어가 안정적으로 얹히는가
6. SDK 공개 계약과 운영 배포가 마무리되는가

## 에이전틱 코딩 운용 규칙

- 구현 에이전트는 한 번에 한 티켓만 소유한다.
- 리뷰 에이전트는 구현 완료 후 `스펙 적합성 -> 코드 품질` 순서로 붙인다.
- 메인 세션은 보드 갱신, 우선순위 유지, 통합 검증만 담당한다.
- 병렬 작업은 반드시 파일 소유 범위를 분리한다.
